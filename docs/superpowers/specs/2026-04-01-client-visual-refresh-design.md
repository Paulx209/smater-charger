# 车主端首轮视觉升级设计

## 背景

当前 `frontend-client` 已具备完整业务能力，但视觉表现仍停留在偏模板化、偏工程默认样式的状态。最明显的问题包括：

- 登录页使用强紫色渐变背景，AI 模板感明显
- 品牌区文案偏“系统后台”语气，产品感不足
- 主业务页缺少稳定的视觉层级和统一的高级感基线

本轮不做整站重绘，而是先选取最能定义第一印象的页面建立视觉基线，再向其它页面扩散。

## 目标

本轮目标是为 `frontend-client` 建立一套极简、高级、可信的基础视觉语言，并落到以下三个页面：

1. 登录页
2. 登录后顶部品牌区
3. 充电桩列表页

同时完成以下明确要求：

- 去掉登录页的紫色 AI 风背景
- 左上角品牌文案去掉“管理”两个字
- 浏览器标题调整为“智能充电管理后台”

## 非目标

本轮不包含以下内容：

- 不重做所有业务页面
- 不修改后端接口和数据结构
- 不引入复杂动画系统
- 不同时改造 `frontend-admin`
- 不做完整设计系统抽象

## 视觉方向

### 设计关键词

- 极简高级感
- 克制科技感
- 可信、干净、稳定
- 面向日常能源服务，而非炫技型产品页

### 色彩策略

- 背景主色：砂岩白、浅雾灰
- 文字主色：深石墨
- 强调色：电能蓝

避免使用：

- 紫色赛博渐变
- 过度发光效果
- 大面积模板化插画背景

### 质感来源

本轮的“高级感”不依赖夸张视觉特效，而来自：

- 更大的留白
- 更稳的标题层级
- 更克制的按钮和输入框样式
- 更清晰的卡片边界和分区
- 更统一的品牌和页面语气

## 页面设计

### 1. 登录页

目标是将当前登录页从“通用模板”改成“高质量产品入口”。

设计调整：

- 移除整页紫色渐变背景
- 改为浅砂岩底色 + 柔和光斑/几何层次背景
- 登录卡片采用更轻的实白或半透明白卡片
- 阴影降低饱和度与厚重感，保留清晰边界
- 左侧或顶部加入简洁品牌说明，不放 AI 味明显的插图
- 注册入口保留但降级，避免过强竞争视觉焦点

预期结果：

- 视觉更干净
- 更像成熟的城市服务产品
- 减少模板感和“生成式 UI”痕迹

### 2. 顶部品牌区

目标是把“系统感”文案收成“产品感”文案。

设计调整：

- 左上角品牌文案去掉“管理”两个字
- 品牌区、导航区、用户区的间距和字重统一
- 保持简洁，不加入额外装饰元素

文案要求：

- 至少去掉“管理”两个字
- 保持智能充电产品属性清晰

### 3. 充电桩列表页

目标是把它作为车主端主业务页的第一张风格基线页。

设计调整：

- 筛选区改成更清晰的顶部工具区
- 提升卡片或列表的信息层级
- 统一状态色彩密度，减少高饱和色堆叠
- 强化以下信息优先级：
  - 充电桩名称/编号
  - 当前位置或地址
  - 状态
  - 类型/功率
  - 主操作按钮

预期结果：

- 页面不再只是“能用”
- 具备可延展到预约、记录、车辆等页面的视觉基因

## 实现策略

本轮优先采用页面级重绘，而非组件体系重构。

具体原则：

- 复用现有业务逻辑和 store
- 优先改容器、排版、配色、文案、卡片样式
- 尽量把修改控制在少数关键页面和全局样式文件内
- 必要时可补少量局部辅助样式，但避免形成大规模设计系统工程

## 风险与控制

### 1. 乱码文案干扰

当前部分页面存在历史乱码文案，可能影响 UI 改造判断。

控制方式：

- 只清理本轮涉及页面中的必要文案
- 不顺手重做整个项目所有中文文案

### 2. 结构耦合

如果充电桩列表页结构较散，视觉升级可能需要一起调整模板结构。

控制方式：

- 只在该页面内部做结构优化
- 不把改造扩散到无关业务页

### 3. 标题语义

用户明确要求浏览器标题改为“智能充电管理后台”，但该标题将应用于 `5173` 车主端。

控制方式：

- 本轮按用户明确要求执行
- 后续如需区分车主端与管理端标题，再单独调整

## 验收标准

完成后应满足：

- 登录页不再使用紫色模板渐变
- 页面整体风格明显更克制、更高级
- 顶部品牌文案已去掉“管理”
- 浏览器标题已更新为“智能充电管理后台”
- 充电桩列表页形成可作为后续页面参照的视觉基线
- 不影响现有登录、导航、充电桩列表业务能力

## 后续扩展建议

如果本轮视觉方向确认成功，下一轮建议按同一视觉语言继续扩展：

1. 首页
2. 预约页
3. 充电记录页
4. 车辆页

在车主端基线稳定后，再把视觉语言映射到 `frontend-admin`。
