# Proposal: 公告模块契约对齐

## Context

当前公告模块在代码层面已经具备：

- 后端公告管理接口
- 后端车主公告接口
- 管理端公告列表与表单页面
- 车主端公告列表、详情与首页公告轮播

但该模块并不真实闭环，原因主要有两类：

1. 前端公告 API 文件把路径写成了 `/api/announcement...`，而两端 Axios `baseURL` 已经配置为 `/api`，存在请求落到 `/api/api/...` 的风险。
2. `frontend-client` 中混入了 `/admin/announcement` 管理页面和管理侧 store 调用，导致端边界不清晰。

这会让公告模块看起来“代码很多”，但实际运行时存在明显契约风险，也会继续污染后续模块规划。

## Change Summary

本 change 只处理公告模块的契约和边界问题：

- 校正前端公告 API 路径，使其与后端 `/announcement` 契约一致
- 校正管理端与车主端的公告调用边界
- 明确车主端只保留消费侧公告能力
- 明确管理端承担公告管理能力

## Non-Goals

- 不修改公告后端业务逻辑
- 不新增公告功能
- 不处理故障报修、费用配置等其他模块
- 不在本 change 中重构整个路由体系
