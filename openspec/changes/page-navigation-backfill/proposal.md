# Proposal: page-navigation-backfill

## Why

当前项目的子页面返回体验不一致。部分页面没有返回入口，部分页面仅依赖 `router.back()`，导致直接访问详情页或刷新后进入页面时缺少稳定的返回路径。

## What Changes

- 为 admin 和 client 端的详情页、表单页、设置页、统计页补齐统一返回入口。
- 引入轻量 `navigateBack` 工具函数，统一处理“优先返回历史，否则跳模块列表”的策略。
- 不改接口、不改业务逻辑，只做页面导航补强。

## Impact

- 提升测试和日常使用时的页面可回退性。
- 降低用户对浏览器后退按钮的依赖。
