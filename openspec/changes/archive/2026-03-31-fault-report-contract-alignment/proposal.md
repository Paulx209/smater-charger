# Proposal: 故障报修契约对齐

## Context

当前故障报修模块的核心问题不是代码缺失，而是代码与真实契约脱节：

- 后端真实接口是 `/fault-report` 与 `/fault-report/admin/*`
- 车主端仍在调用 `/api/fault-reports`、`/my`、`/upload`、`/stats/pile/*`
- 车主端页面依赖了后端并不存在的字段，例如 `faultType`、`images`、`processTime`、`resolveTime`
- 管理端虽然已有后端接口，但没有对应的前端管理闭环

继续在当前基础上叠加开发，只会把“看起来有功能、实际链路不通”的状态固化下来。

## Change Summary

本 change 只做真实闭环恢复：

- 以前端对齐后端既有故障报修契约为第一原则
- 收缩车主端到最小可用模型
- 建立管理端列表、详情处理、统计闭环
- 清理前端对后端不存在字段与端点的依赖

## Non-Goals

- 不新增故障分类体系
- 不新增图片上传
- 不新增扩展时间线字段
- 不扩展后端实体结构
- 不做跨模块重构
