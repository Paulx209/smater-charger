# 智能充电桩管理系统 - 项目功能盘点与待实现差异清单

## 文档说明

本文档已从“纯待办列表”调整为“代码基线盘点表”。

自 `2026-03-30` 起，本文档的判断规则如下：

- 以代码为准，不以旧规划文档为准
- 按一级业务模块盘点，而不是按仓库目录盘点
- 同时对照 `backend`、`frontend-admin`、`frontend-client`
- `openspec/` 仅用于描述后续具体变更，不再承担全量现状记录职责

状态说明：

- `已实现`：该端已有明确代码实现，且路径、页面或接口完整可追踪
- `部分实现`：已有代码，但仍存在断点、契约不一致或只覆盖部分能力
- `未实现`：该端没有对应功能入口或没有可用实现
- `不适用`：该模块当前不要求该端承担完整能力

## 三、与旧文档相比的主要偏差

以下结论已经由代码确认，旧文档需要停止继续沿用原判断：

1. `费用配置管理` 不是待开发状态，后端与管理端已经实现，车主侧也已有费用展示与预估。
2. `充电记录管理` 不是待开发状态，后端与车主端已经实现，且已有 OpenSpec 变更记录。
3. `占位预警/预警通知` 不是待开发状态，后端和车主端已经实现基础能力。
4. `故障报修` 不是“无实现”，而是“后端已实现、车主端代码存在但契约未对齐、管理端缺页”。
5. `系统公告` 已完成契约校正，不再属于待开发或待校正模块。
6. `用户管理` 与 `数据统计` 已经在管理端实现，不应继续列为基础待开发功能。
